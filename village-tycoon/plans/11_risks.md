# 11. Риски и митигация

## Технические риски

### R1. Хостинг на ПК — недоступность
**Вероятность**: высокая
**Импакт**: средний (для пет-проекта), высокий (для роста аудитории)

**Описание**: твой ПК выключен/перезагружен/без интернета → Mini App недоступен → все тестеры видят ошибку.

**Митигация**:
- Cloudflare Tunnel в режиме `--retries 999 --autorestart`
- Автозапуск Docker Compose при загрузке системы (`docker compose up -d` через Task Scheduler)
- Health-check бот посылает раз в минуту запрос на свой URL, если падает → пишет в служебный канал
- При планируемом ребуте — заранее предупредить тестеров
- **Долгосрочно**: переезд на VPS Hetzner ($5-10/мес) когда наберётся 100+ активных игроков

### R2. Telegram меняет правила Mini Apps
**Вероятность**: низкая, но болезненная

**Описание**: Telegram внезапно ограничивает API, меняет HMAC схему, банит твой бот.

**Митигация**:
- Архитектура совместима с обычным веб-приложением (auth можно переключить)
- Не используем экспериментальные API
- Регулярно проверяем changelog Telegram Bot API
- Бэкап аккаунтов: храним email связку для каждого юзера для возможной миграции

### R3. Postgres переполнен на бесплатном тире
**Вероятность**: средняя при росте

**Описание**: `villager_event_log` растёт быстро, диск заканчивается.

**Митигация**:
- Партиции по дате с auto-drop старых (>90 дней) для event_log
- Резервное копирование на внешний диск
- Мониторинг занимаемого места, alert при 80%

### R4. Race conditions при concurrent build/upgrade
**Вероятность**: средняя

**Описание**: игрок быстро тапает дважды → две постройки за одни ресурсы. Или: одна транзакция применилась, вторая — нет, клиент в рассинхроне.

**Митигация**:
- `SELECT ... FOR UPDATE` на ресурсах в транзакции
- Идемпотентные ключи `X-Idempotency-Key` на чувствительных endpoints
- Серверные локи на 100ms на user-level операции
- При desync — возврат полного state в ответе

### R5. Phaser 3 падает на старых девайсах
**Вероятность**: средняя на Android < 9

**Описание**: WebGL не поддерживается, или игра тормозит.

**Митигация**:
- Phaser auto-fallback на Canvas2D рендер
- Проверять `window.WebGLRenderingContext` и предупреждать
- Спрайтшиты вместо отдельных PNG (меньше batch вызовов)
- Целевой минимум: Android 9 / iOS 13

## Контентные риски

### R6. Ассеты не успевают за кодом
**Вероятность**: высокая

**Описание**: я пишу систему построек, а у нас 0 PNG зданий → играть нечем.

**Митигация**:
- Placeholder-цветные квадраты с текстом "LUMBERMILL LV1" пока ассета нет
- Naming convention строгий — замена 1:1 без правок кода
- Параллельная работа: генерация во время разработки систем
- MVP-список ассетов (~70 PNG) реально сделать за неделю

### R7. Балансные числа из GDD не работают
**Вероятность**: очень высокая

**Описание**: 100 дерева/час слишком мало или слишком много, прогресс ломается.

**Митигация**:
- Баланс в JSON, правится без передеплоя
- A/B тесты на 50/50 группах в Beta
- Метрики геймплея (как быстро доходят до эпохи 2, среднее время сессии)
- Готовность переписать формулы 5-10 раз

### R8. Контента не хватает / фичи запросов неожиданно
**Вероятность**: высокая

**Описание**: тестеры играют 2 дня и упираются в "всё построил, делать нечего".

**Митигация**:
- Daily quests дают вечную причину возвращаться
- Глубокий апгрейд цикл (каждое здание имеет 50 уровней в полном GDD)
- Переход эпох даёт новый контент
- Открытый список "что добавить дальше" по фидбэку игроков

## Геймплейные риски

### R9. Идл-цикл не цепляет
**Вероятность**: средняя

**Описание**: "ну собрал ресурсы, ну построил, скучно".

**Митигация**:
- Daily квесты с вариативными целями
- Сезонные ивенты (даже маленькие)
- Социальные элементы (даже просто "посмотреть деревню друга")
- Лидерборды, ачивки
- **Главное** — приятная анимация и звук получения наград (dopamine)

### R10. Игроки эксплуатируют баланс
**Вероятность**: средняя

**Описание**: находят "глитч" — например, выгодная цепочка крафта дающая бесконечное золото.

**Митигация**:
- Лог всех экономических событий
- Регулярный аудит топ-10 игроков по богатству — что они делают
- Soft-cap на ежедневный прирост ресурсов
- Анти-чит rate limits

### R11. Боты и фарм-аккаунты
**Вероятность**: средняя на Beta+

**Описание**: автоматизированные клиенты собирают ресурсы 24/7.

**Митигация**:
- Серверная валидация всех действий
- Rate limits per-user (Redis-based)
- Поведенческий анализ (точность интервалов, длительность сессий)
- Капча при подозрении
- Soft-ban → permanent ban при подтверждении

## Риски процесса разработки

### R12. Bus factor 1 (один разработчик)
**Вероятность**: реальность

**Описание**: всё держится на Claude Code и тебе. Если кто-то выключился — стоп.

**Митигация**:
- Хорошая документация (этот план!)
- Читаемый код с комментариями WHY
- Все решения — в md-файлах
- Регулярные коммиты с осмысленными сообщениями
- Open-source стек (нет вендорлока)

### R13. Скоп-крип / постоянное расширение
**Вероятность**: высокая

**Описание**: GDD большой, соблазн добавить ещё одну фичу пока делаем.

**Митигация**:
- Жёсткий MVP scope (см. [03_mvp_scope.md](03_mvp_scope.md))
- Любая новая идея → бэклог, не текущая работа
- Регулярная переоценка приоритетов

### R14. Геймдиз+разработчик в одном лице (ты)
**Вероятность**: реальность

**Описание**: ты и решаешь что делать, и проверяешь как сделано. Нет независимой проверки качества.

**Митигация**:
- Тестеры (5-10 человек) с самого MVP
- Регулярные плейтесты
- Логи и аналитика как объективная оценка

## Финансовые риски

### R15. Хостинг становится дорогим при росте
**Вероятность**: проблема при ≥10к DAU

**Описание**: Postgres + Redis + WebSocket на 10к concurrent — нужен серьёзный хостинг.

**Митигация**:
- Кеширование агрессивное (Redis hot data)
- Партиции в Postgres
- Rate limits жёсткие
- Telegram Stars доход покроет хостинг при 1-2% conversion

### R16. Telegram Stars комиссия 30%
**Вероятность**: реальность

**Описание**: из каждой покупки 30% забирает Telegram.

**Митигация**:
- Это просто учитывается в ценообразовании
- Альтернативы (свой billing) — нарушение TOS, забанят

### R17. RMT (real-money trading) подрывает экономику
**Вероятность**: средняя при росте

**Описание**: игроки продают ресурсы за реальные деньги вне игры.

**Митигация**:
- Permanent ban обоих участников
- Отслеживание подозрительных трансферов на аукционе
- Новые аккаунты не могут торговать первые 7 дней

## Юридические риски

### R18. GDPR / персональные данные
**Вероятность**: низкая (Telegram сам данные собирает)

**Митигация**:
- Не хранить PII сверх того что даёт Telegram (tg_id + name)
- Возможность удалить аккаунт по запросу
- Privacy policy в боте

### R19. Возрастные ограничения
**Вероятность**: проблема при наличии Stars (gambling-like)

**Митигация**:
- 13+ возрастной маркер в Telegram Bot
- Loot-боксы прозрачные (показываем шансы)
- Никакого нативного gambling
