# 10. Roadmap

Поэтапный план разработки. Каждый этап — рабочий билд, который **играбелен** к концу этапа. Не ждём конца всего, чтобы что-то увидеть.

## Этап 0 — Planning ✅ (текущий)

**Длительность**: 1-2 дня
**Статус**: в процессе

- [x] Прочитать GDD
- [x] Аудит реалистичности
- [x] Выбрать стек (Phaser 3 + FastAPI + Postgres + Redis)
- [x] Определить MVP scope
- [x] Написать план-файлы
- [ ] Согласовать стек с пользователем
- [ ] Сгенерировать первые 14 ассетов (для запуска)

**Выход**: репозиторная папка `village-tycoon/` со всеми планами + первые ассеты на старт.

---

## Этап 1 — Foundation (3-5 дней)

**Цель**: пустая, но рабочая инфраструктура. Игра открывается, авторизуется, показывает пустую карту.

### 1.1 Backend инфраструктура (1-2 дня)
- [ ] `app/villager/` модуль создан
- [ ] `app/villager/audit.py` — миграции БД
- [ ] `app/api/villager_api.py` — каркас FastAPI router
- [ ] Endpoint `GET /api/villager/state` — возвращает пустого юзера
- [ ] Endpoint `GET /api/villager/config` — возвращает MVP-конфиги
- [ ] Auth через Telegram initData (переиспользуем `app/api/auth.py`)
- [ ] Конфиг buildings/resources/quests в JSON-файлах
- [ ] Логирование через Sentry

### 1.2 Frontend каркас (1-2 дня)
- [ ] `village-tycoon/` Vite проект, TS+Phaser+Solid
- [ ] `BootScene` — загрузка ассетов через JSON-манифест
- [ ] `MapScene` — рендер пустой карты 16×16, тайлы placeholder
- [ ] Pan + zoom камера (pinch-zoom для мобилы)
- [ ] `UIScene` overlay — placeholder top bar и bottom nav
- [ ] Solid-компонент для модалки
- [ ] API клиент с правильным auth header

### 1.3 Docker Compose (1 день)
- [ ] `docker-compose.yml` с postgres, redis, backend, frontend
- [ ] Cloudflare Tunnel настройка
- [ ] Бот хендлер `/villager` → ссылка на туннель

**Выход**: открываешь `/villager` в Telegram, видишь пустую зелёную карту с ресурсной панелью наверху.

---

## Этап 2 — MVP Core (1-2 недели)

**Цель**: играть можно. Минимальный циклический фан.

### 2.1 Постройка зданий (2-3 дня)
- [ ] Drag-and-drop здания из меню Build на тайл
- [ ] Сетка отображается при размещении
- [ ] Серверная валидация координат
- [ ] Транзакция: списание ресурсов + INSERT здания
- [ ] Анимация появления здания (pop-in)
- [ ] Состояние "стройка" с таймером
- [ ] Эффект scaffolding на зданиях в стройке

### 2.2 Idle-механика (1-2 дня)
- [ ] Lazy compute идл-дохода
- [ ] Кап склада соблюдается
- [ ] Эффективность офлайна (50%)
- [ ] Модалка "Пока тебя не было" при возврате
- [ ] Кнопка "Забрать всё" собирает с всех зданий
- [ ] Анимация летящих ресурсов в счётчики

### 2.3 Апгрейды (1-2 дня)
- [ ] Тап на здание → модалка с инфой
- [ ] Кнопка "Проапгрейдить" с ценой и временем
- [ ] Серверная валидация и списание
- [ ] Визуальная подмена спрайта на следующий уровень после завершения
- [ ] Перемещение здания (drag) — бесплатно
- [ ] Снос здания с возвратом 50%

### 2.4 Квесты (1-2 дня)
- [ ] 10 базовых квестов в JSON
- [ ] Прогресс отслеживается на сервере при каждом значимом действии
- [ ] UI-панель квестов
- [ ] Награды забираются по нажатию
- [ ] Цепочка квестов раскрывается по мере завершения

### 2.5 Уведомления через бота (1 день)
- [ ] При завершении стройки — push в Telegram через бот
- [ ] `/status` команда работает
- [ ] `/collect` команда работает
- [ ] Redis pub/sub между backend и bot

### 2.6 Полировка MVP (1-2 дня)
- [ ] Все 8 зданий × 3 уровня имеют ассеты
- [ ] Все 6 ресурсов имеют иконки
- [ ] UI кнопки и панели имеют ассеты
- [ ] Эффекты при завершении квеста, постройки, апгрейда
- [ ] Звук **отключён** (без него лучше для MVP)

**Выход**: 30-60 минут игры от первого тапа до первого апгрейда. Можно дать 5 тестерам.

## Этап 3 — Beta (4-6 недель)

**Цель**: расширенный билд. Социалка. Монетизация (опционально).

### 3.1 Эпохи 3-4 (1-2 недели)
- [ ] Дополнительные ресурсы: уголь, медь, олово, железо, шкура, шерсть, лён, бронза, кирпичи, доски, кожа, ткань
- [ ] +20 зданий: шахты, кузница, фермы животных, ткацкая, дубильня, кирпичная печь
- [ ] Требования эпохи: достаточный уровень Town Hall, выполненные квесты
- [ ] Анимация перехода в новую эпоху

### 3.2 Гильдии (1-2 недели)
- [ ] Создание гильдии (10к золота)
- [ ] Гильдийный чат через WebSocket
- [ ] Список членов, роли (leader/officer/member)
- [ ] Гильдийные квесты (общие, обновляются еженедельно)
- [ ] Лидерборд гильдий

### 3.3 Аукцион P2P (1 неделя)
- [ ] Выставить лот (предмет/ресурс)
- [ ] Просмотр активных лотов
- [ ] Ставки + buy-now
- [ ] Автозавершение через cron-задачу
- [ ] История моих сделок

### 3.4 Базовое дерево технологий (3-5 дней)
- [ ] 10-15 технологий, разбитых на 3-4 ветки
- [ ] Изучение требует ресурсы и время
- [ ] Эффекты технологий (например, +10% дереву от лесопилок)

### 3.5 Аналитика и метрики (3 дня)
- [ ] Event log пишется в `villager_event_log`
- [ ] Dashboard в Grafana с DAU, постройками, ресурсами
- [ ] Воронка онбординга

### 3.6 Telegram Stars (2-3 дня)
- [ ] Подключение Stars API через бота
- [ ] Магазин гемов
- [ ] Идемпотентность транзакций
- [ ] Проверка чарджбэков

### 3.7 Локализация (2 дня)
- [ ] EN, RU
- [ ] i18n библиотека на фронте
- [ ] Локали в JSON

**Выход**: можно дать 50-500 тестерам, монетизация работает.

**Цель**: всё открыто, маркетинг, Live ops.

- [ ] Несколько гильдий в альянс

### 5.4 Сезонный пасс
- [ ] Прогресс через активность

- [ ] 8 эпохальных чудес (по одному на эру)

- [ ] Глобальные бонусы всем гильдиям

- Сезон каждые 30-45 дней (новый ивент, новый пасс)

- Глобальный мегаивент раз в 6 мес
- Расширения: морская торговля, новые регионы

## Критерии перехода между этапами

| От → К | Критерий |
|---|---|
| 0 → 1 | План утверждён, первые ассеты есть |
| 1 → 2 | Карта рендерится, авторизация работает |
| 2 → 3 | 5 тестеров играют 30+ мин без багов |
| 3 → 4 | DAU стабилизировался, монетизация ≥1% conversion |
| 4 → 5 | Soft launch metrics ≥ targets |
| 5 → ∞ | Production stable, customer support active |

## Что может ускорить разработку

- **Ассеты впереди кода** — меньше переделок
- **Ясные конфиги** — баланс правится без редеплоя
- **Жёсткий MVP** — не уплываем в фичи

## Что может замедлить

- **Перфекционизм над визуалом** — итерации арта
- **Балансные правки** — 100 раз меняется
- **Неправильная архитектура** — переделка всего после
- **Telegram API изменения** — редко, но больно
