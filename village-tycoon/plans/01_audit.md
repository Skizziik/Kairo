# 01. Аудит GDD

Документ оценивает реалистичность исходного GDD (`Village_Tycoon_GDDv3.docx`) для нашего флоу: один разработчик (Claude Code) + один геймдиз (пользователь).

## Объём GDD в цифрах

| Параметр | Значение в GDD |
|---|---|
| Эпохи | 8 |
| Постройки | 120+ (до 50 уровней каждая) |
| Ресурсы | 42 |
| Предметы | 300+ |
| Рецепты крафта | 250+ |
| Технологии | 200+ |
| Юниты армии | 35 |
| Квесты | 100+ сюжетных + бесконечные daily/weekly |
| Достижения | 150+ |
| Сезонные ивенты | 12/год + 6 разовых |
| Региональные карты | 10 + 1 каждый сезон |

GDD оценивает разработку как **6-12 месяцев работы команды из 3-5 человек** (Claude Code + геймдиз + художник + UI/UX + DevOps).

## Реалистичная оценка

### ✅ Что реально для нашего флоу
- **MVP** (5-8 зданий, эпохи 1-2, idle-доход, базовые квесты) — 2-4 недели
- **Beta** (+гильдии, аукцион, эпохи 3-4) — ещё 4-6 недель
- **Soft Launch** (битвы, биржа, эпохи 5-6) — ещё 6-10 недель
- **Полный GDD** — 6-12 месяцев минимум, при условии что геймдиз+ассеты не отстают

### ⚠️ Что вызывает сомнения
- **Real-time WebSocket** для чата гильдии и live-биржи — на хостинге локального ПК работает, на Render free — нет
- **ML anti-cheat** — это уровень Supercell. У нас будет серверная валидация и rate-limits. Этого достаточно для масштаба пет-проекта
- **Telegram Stars** — требует подключения биллинга, верификации в Telegram. Откладываем до Beta
- **Балансные числа** в GDD — это **первый драфт**. Они почти наверняка нерабочие на практике, нужны итерации
- **Качество ассетов "как app store"** — узкое горлышко. Phaser сам по себе не делает игру красивой, всё решает арт. Полагаемся на пользователя

### ❌ Что вырезаем или сильно урезаем
- **PvP** — отложено до Soft Launch, в MVP/Beta нет
- **Альянсы (несколько гильдий)** — отложено до post-launch
- **Полный технологический трекер 200+ техник** — в MVP только 5-10 базовых техник, не дерево
- **Региональные карты** — в MVP одна карта, остальное позже
- **Сезонные пассы** — после Beta
- **Реклама** — после Beta

## Ключевые риски GDD

| Риск | Вероятность | Митигация |
|---|---|---|
| Объём контента блокирует прогресс (300 предметов сами по себе не появятся) | Высокая | Урезанный MVP, расширение по этапам |
| Ассеты не успевают за кодом | Высокая | Ассет-пайплайн жёстко стандартизован, генерим минимум для MVP |
| Балансные числа из GDD не работают | Очень высокая | Числа в JSON-конфигах, итерируем без передеплоя |
| Хостинг на ПК — нестабильность | Средняя | Cloudflare Tunnel + автозапуск Docker, режим "только когда онлайн" |
| Telegram меняет правила Mini App | Низкая | Архитектура совместима с обычным веб-приложением |
| Один разработчик = bus factor 1 | Высокая | Хорошая документация, читаемый код, всё в репо |

## Решение

**Идём от MVP**, постепенно расширяем. Полный GDD — это компас, не план на завтра. Каждые 2 недели — играбельный билд + правки баланса.

Не пытаемся реализовать всё сразу. Не пытаемся идеально первым подходом — итерируем.
